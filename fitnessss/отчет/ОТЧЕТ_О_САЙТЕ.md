# ПОДРОБНЫЙ ОТЧЕТ О САЙТЕ "ФИТНЕС-ЦЕНТР"

## 1. ОБЩЕЕ ОПИСАНИЕ САЙТА

### 1.1. Назначение
Информационная система "Фитнес-центр" — это веб-приложение для управления фитнес-центром, предназначенное для:
- Предоставления информации об услугах, тренерах и абонементах клиентам
- Оформления и оплаты заказов онлайн
- Управления контентом и заказами администратором
- Отслеживания истории покупок пользователями

### 1.2. Технологический стек
- **Backend**: PHP 7.4+
- **Frontend**: HTML5, CSS3, JavaScript (ES6+)
- **Фреймворки**: Bootstrap 5.3.0
- **База данных**: MySQL 5.7+ (InnoDB)
- **Сервер**: Apache/Nginx с поддержкой PHP
- **Библиотеки**: Font Awesome 6.4.0 (иконки)

### 1.3. Архитектура
Сайт построен по модульной архитектуре:
- **MVC-подобная структура**: разделение логики, представления и данных
- **Компонентный подход**: переиспользуемые компоненты (header, footer)
- **RESTful API**: эндпоинты для обработки запросов
- **AJAX**: асинхронная обработка форм и данных

## 2. СТРУКТУРА БАЗЫ ДАННЫХ

### 2.1. Таблицы

#### 2.1.1. `users` (Пользователи)
- **Назначение**: Хранение данных пользователей системы
- **Поля**:
  - `id` - Уникальный идентификатор
  - `username` - Логин пользователя (уникальный)
  - `email` - Email адрес (уникальный)
  - `password` - Хешированный пароль
  - `full_name` - Полное имя пользователя
  - `phone` - Номер телефона
  - `role` - Роль (user/admin)
  - `created_at` - Дата регистрации
  - `updated_at` - Дата последнего обновления

#### 2.1.2. `trainers` (Тренеры)
- **Назначение**: Информация о тренерах фитнес-центра
- **Поля**:
  - `id` - Уникальный идентификатор
  - `full_name` - Полное имя тренера
  - `specialization` - Специализация
  - `experience` - Опыт работы (в годах)
  - `phone` - Контактный телефон
  - `email` - Email адрес
  - `photo` - Путь к фотографии
  - `bio` - Биография/описание
  - `schedule` - Расписание работы

#### 2.1.3. `services` (Услуги)
- **Назначение**: Каталог услуг фитнес-центра
- **Поля**:
  - `id` - Уникальный идентификатор
  - `name` - Название услуги
  - `description` - Описание услуги
  - `price` - Цена услуги
  - `duration` - Длительность в минутах
  - `category` - Категория (Групповые занятия, Персональные тренировки)
  - `trainer_id` - Связь с тренером (внешний ключ)
  - `schedule` - Расписание проведения
  - `max_participants` - Максимальное количество участников
  - `is_active` - Активна ли услуга

#### 2.1.4. `subscriptions` (Абонементы)
- **Назначение**: Типы абонементов
- **Поля**:
  - `id` - Уникальный идентификатор
  - `name` - Название абонемента
  - `description` - Описание
  - `price` - Цена абонемента
  - `duration_days` - Срок действия в днях
  - `visits_count` - Количество посещений (NULL = безлимит)
  - `services_included` - Включенные услуги
  - `is_active` - Активен ли абонемент

#### 2.1.5. `orders` (Заказы)
- **Назначение**: Заказы пользователей
- **Поля**:
  - `id` - Уникальный идентификатор заказа
  - `user_id` - Связь с пользователем
  - `total_amount` - Общая сумма заказа
  - `status` - Статус заказа (pending, confirmed, cancelled, completed)
  - `order_date` - Дата создания заказа
  - `payment_method` - Способ оплаты
  - `payment_status` - Статус оплаты (unpaid, paid, refunded)
  - `notes` - Дополнительные заметки

#### 2.1.6. `order_items` (Элементы заказа)
- **Назначение**: Детализация заказов (товары/услуги в заказе)
- **Поля**:
  - `id` - Уникальный идентификатор
  - `order_id` - Связь с заказом
  - `item_type` - Тип товара (service/subscription)
  - `item_id` - ID услуги или абонемента
  - `quantity` - Количество
  - `price` - Цена за единицу
  - `subtotal` - Итоговая сумма позиции

#### 2.1.7. `user_subscriptions` (Активные абонементы пользователей)
- **Назначение**: Хранение активных абонементов пользователей
- **Поля**:
  - `id` - Уникальный идентификатор
  - `user_id` - Связь с пользователем
  - `subscription_id` - Связь с типом абонемента
  - `order_id` - Связь с заказом
  - `start_date` - Дата начала действия
  - `end_date` - Дата окончания действия
  - `visits_used` - Количество использованных посещений
  - `is_active` - Активен ли абонемент

### 2.2. Связи между таблицами
- `services.trainer_id` → `trainers.id` (многие к одному)
- `orders.user_id` → `users.id` (многие к одному)
- `order_items.order_id` → `orders.id` (многие к одному)
- `user_subscriptions.user_id` → `users.id` (многие к одному)
- `user_subscriptions.subscription_id` → `subscriptions.id` (многие к одному)
- `user_subscriptions.order_id` → `orders.id` (многие к одному)

### 3.3. Корзина и покупки

#### 3.3.1. Корзина (`cart.php`)
**Возможности**:
- Просмотр товаров в корзине
- Отображение для каждого товара:
  - Название
  - Тип (услуга/абонемент)
  - Цена за единицу
  - Количество
  - Итоговая стоимость позиции
- Изменение количества товаров
- Удаление товаров из корзины
- Подсчет общей суммы заказа
- Очистка корзины
- Оформление заказа
- Хранение корзины в localStorage браузера

#### 3.3.2. Оформление заказа (`checkout.php`)
**Процесс**:
1. **Создание заказа**:
   - Автоматический расчет общей суммы
   - Создание записи в таблице `orders`
   - Сохранение элементов заказа в `order_items`
   - Очистка корзины

2. **Форма оплаты**:
   - Отображение информации о заказе
   - Форма ввода данных плательщика:
     - ФИО
     - Email
     - Телефон
     - Адрес (опционально)
   - Форма ввода данных карты:
     - Номер карты (с автоматическим форматированием)
     - Месяц и год действия
     - CVC/CVV код
   - Валидация всех полей
   - Кнопка "Оплатить"

3. **Обработка оплаты**:
   - Отправка данных через AJAX
   - Обновление статуса заказа на "Оплачено"
   - Создание активных абонементов (если куплены)
   - Показ окна "Спасибо за покупку"
   - Перенаправление на страницу "Мои заказы"

#### 3.3.3. Мои заказы (`orders.php`)
**Возможности**:
- Просмотр истории всех заказов пользователя
- Информация о каждом заказе:
  - Номер заказа
  - Дата создания
  - Количество товаров
  - Общая сумма
  - Статус заказа (Ожидает, Подтвержден, Оплачено, Отменен, Завершен)
  - Статус оплаты (Не оплачен, Оплачен, Возврат)
- Просмотр деталей заказа:
  - Список всех товаров в заказе
  - Тип каждого товара
  - Количество
  - Цена
  - Итоговая сумма
- Сортировка по дате (новые сверху)

### 3.4. Профиль пользователя

#### 3.4.1. Профиль (`profile.php`)
**Возможности**:
- **Просмотр информации**:
  - Логин (неизменяемый)
  - Email
  - Полное имя
  - Телефон
  - Роль
  - Дата регистрации
  - Дата последнего обновления

- **Редактирование данных**:
  - Изменение email (с проверкой уникальности)
  - Изменение полного имени
  - Изменение телефона
  - Изменение пароля (опционально):
    - Новый пароль
    - Подтверждение пароля
    - Минимум 6 символов
  - Сохранение изменений

- **Активные подписки**:
  - Список всех активных абонементов
  - Информация о каждом:
    - Название абонемента
    - Дата окончания действия
    - Количество использованных/доступных посещений (если применимо)

### 3.5. Дополнительные возможности

#### 3.5.1. Отображение активной подписки
- В шапке сайта рядом с ником пользователя отображается бейдж с названием активной подписки
- Показывается только если:
  - Подписка активна (`is_active = 1`)
  - Дата окончания еще не наступила (`end_date >= CURDATE()`)
- При наведении показывается подсказка с полным названием и датой окончания

#### 3.5.2. Счетчик корзины
- В навигации отображается количество товаров в корзине
- Обновляется автоматически при добавлении/удалении товаров
- Хранится в localStorage

#### 3.5.3. Уведомления
- Всплывающие уведомления о действиях:
  - Добавление товара в корзину
  - Удаление товара из корзины
  - Очистка корзины
  - Успешная оплата
  - Ошибки при операциях

## 5. ПРОЦЕСС ПОКУПКИ (ПОЛНЫЙ ЦИКЛ)

### 5.1. Добавление в корзину
1. Пользователь просматривает услуги или абонементы
2. Нажимает кнопку "Добавить в корзину"
3. Товар сохраняется в localStorage браузера
4. Обновляется счетчик корзины в навигации
5. Показывается уведомление об успешном добавлении

### 5.2. Оформление заказа
1. Пользователь переходит в корзину
2. Проверяет список товаров и количество
3. Может изменить количество или удалить товары
4. Нажимает "Оформить заказ"
5. Создается заказ в базе данных:
   - Запись в таблице `orders` со статусом "pending" и оплатой "unpaid"
   - Записи в таблице `order_items` для каждого товара
6. Корзина очищается
7. Пользователь перенаправляется на страницу оплаты

### 5.3. Оплата заказа
1. На странице оплаты отображается:
   - Номер заказа
   - Список товаров с количеством и ценами
   - Общая сумма к оплате
2. Пользователь заполняет форму:
   - Данные плательщика (ФИО, Email, Телефон, Адрес)
   - Данные карты (Номер, Месяц, Год, CVC/CVV)
3. При нажатии "Оплатить":
   - Валидация всех полей
   - Отправка данных через AJAX на `api/process_payment.php`
   - Обновление заказа:
     - `payment_status` = 'paid'
     - `status` = 'confirmed'
     - `payment_method` = 'card'
     - Сохранение последних 4 цифр карты в notes
4. Создание активных абонементов:
   - Если в заказе есть абонементы, создаются записи в `user_subscriptions`
   - Устанавливается дата начала (текущая дата)
   - Рассчитывается дата окончания (start_date + duration_days)
   - Абонемент помечается как активный
5. Показывается окно "Спасибо за покупку"
6. После закрытия окна - перенаправление на "Мои заказы"

### 5.4. Результат покупки
- Заказ отображается в "Мои заказы" со статусом "Оплачено"
- Если куплен абонемент:
  - Он появляется в профиле пользователя в разделе "Активные подписки"
  - Отображается в шапке сайта рядом с ником пользователя
  - Будет виден до истечения срока действия

### 6.1. Защита от SQL-инъекций

## 11. ПРОЦЕССЫ И БИЗНЕС-ЛОГИКА

### 11.1. Процесс покупки услуги
1. Пользователь выбирает услугу
2. Добавляет в корзину
3. Оформляет заказ
4. Оплачивает заказ
5. Заказ получает статус "Оплачено"
6. Пользователь может использовать услугу

### 11.2. Процесс покупки абонемента
1. Пользователь выбирает абонемент
2. Добавляет в корзину
3. Оформляет заказ
4. Оплачивает заказ
5. Автоматически создается активная подписка:
   - Устанавливается дата начала (текущая дата)
   - Рассчитывается дата окончания
   - Подписка становится активной
6. Подписка отображается в профиле и в шапке сайта
7. Подписка действует до истечения срока

### 11.3. Управление статусами заказов
- **pending** - Заказ создан, ожидает оплаты
- **confirmed** - Заказ подтвержден (после оплаты)
- **cancelled** - Заказ отменен
- **completed** - Заказ завершен

### 11.4. Управление статусами оплаты
- **unpaid** - Не оплачен
- **paid** - Оплачен
- **refunded** - Возврат средств
